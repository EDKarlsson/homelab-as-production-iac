Infrastructure as Code for a Proxmox homelab running a K3s Kubernetes cluster. Everything from VM provisioning to application deployment is managed through code.

## Cluster Overview

```
Proxmox Cluster (5 nodes, 10.0.0.0/24)
├── K3s Control Plane (3 servers)
│   ├── node-02   — 4 cpu / 4 GB   / 10.0.0.50
│   ├── node-03  — 4 cpu / 4 GB   / 10.0.0.51
│   └── node-04  — 4 cpu / 4 GB   / 10.0.0.52
├── K3s Workers (5 agents)
│   ├── node-02    — 6 cpu / 26 GB  / 10.0.0.62
│   ├── node-03  — 10 cpu / 24 GB / 10.0.0.63
│   ├── node-04  — 8 cpu / 24 GB  / 10.0.0.64
│   ├── node-01       — 12 cpu / 24 GB / 10.0.0.60
│   └── node-05      — 10 cpu / 60 GB / 10.0.0.61
├── PostgreSQL HA (K3s datastore + app databases)
│   ├── Primary (node-01)  — 2 cpu / 4 GB / 10.0.0.45
│   ├── Standby (node-05) — 2 cpu / 4 GB / 10.0.0.46
│   └── VIP (keepalived)       — 10.0.0.44
└── Storage
    └── Synology NAS             — NFS at 10.0.0.161
```

- **K3s v1.32.12+k3s1** on Ubuntu 24.04 VMs (cloud-init provisioned)
- **External PostgreSQL** datastore with async streaming replication and keepalived VIP failover
- **1Password Connect** (HA) for secrets management across Terraform, Ansible, and Kubernetes

## Quick Start

```bash
# Prerequisites: terraform, kubectl, ansible, 1Password Connect token

# Source environment
source .env.d/terraform.env

# Deploy infrastructure
cd infrastructure
terraform init
terraform apply          # Creates 9 VMs + 2 PG HA VMs (~30 min)

# Provision K3s cluster
cd ../ansible
ansible-playbook -i inventory/k3s.yml playbooks/k3s-cluster.yml --forks=1

# Verify cluster
export KUBECONFIG=~/.kube/config-homelab
kubectl get nodes        # 8 nodes Ready
scripts/k8s/k3s-verify.sh all   # 46/46 checks

# Bootstrap FluxCD (deploys all platform services + apps)
flux bootstrap github --path=clusters/homelab
```

## CI/CD Quick-Fail Triage

`main` is protected with required status check: `CI Summary`.

- Check PR statuses: `gh pr checks <pr-number>`
- Inspect failed run logs: `gh run view <run-id> --log-failed`
- Re-run Kubernetes/Flux checks locally: `bash scripts/k8s/validate.sh`
- Re-run policy checks locally: `bash scripts/ci/policy-check.sh`
- Re-run Ansible syntax checks locally:
  - `ansible-playbook -i ansible/inventory/k3s.yml ansible/playbooks/k3s-cluster.yml --syntax-check`
  - `ansible-playbook -i ansible/inventory/k3s.yml ansible/playbooks/k3s-upgrade.yml --syntax-check -e k3s_version=v1.32.12+k3s1`
  - `ansible-playbook -i ansible/inventory/k3s.yml ansible/playbooks/pg-backup.yml --syntax-check`
